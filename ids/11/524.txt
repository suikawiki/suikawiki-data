[2]
>>1 は仕様書ではなくメモのようなので、曖昧なところがあるのは仕方がないでしょう。
でも別の実装の登場に期待する意味で、もう少し厳密な定義が欲しい。
という観点から疑問点をリストアップ。

[3]
'''記法が認識される場所'''。見出しを表す [CODE[*]]
やリストを表す [CODE[-]] は行頭でのみその意味を持つのでしょうか。
リンクや画像を表す [CODE['''[''']]・・・[CODE[''']''']] 
などは行頭以外でも使えるのでしょうか。
画像のテキスト部で記法は認識されるのでしょうか。

認識されない場所ではそれはどう処理されるのでしょうか。
HTML タグはどうなるのでしょうか。 [CODE[&]]
はどう扱われるのでしょうか。

HTML タグではない [CODE[<]] はどう扱われるのでしょうか。

[7]
'''改行の効果'''。見出しやリスト項目の行の最後で見出しやリスト項目は終わるのでしょうか。

[8] 
'''段落前後の改行'''。空行は改段落になりますが、
文書の先頭や末尾、見出し・リスト等の直前や直後ではどうなるのでしょうか。

[4] 
'''アンカーとして使える文字'''。
見出しのアンカーとして使える文字の種類に制限はあるのでしょうか。
- [[HTML]]の制限に従うのでしょうか。
-- [CODE(HTMLa)@en[[[name]]]][[属性]]でしょうか、
[CODE(HTMLa)@en[[[id]]]][[属性]]でしょうか。
-- どの版の[[HTML]]でしょうか。
- 幸之介記法としての制限はないのでしょうか。
- [CODE['''[''']] や [CODE[''']''']] は認められるのでしょうか。
- [[改行]]は認められるのでしょうか。 (>>3)

[5]

> 見出しリンクが張られていたら、idアンカーはなくなる

意味がよくわかりません。

[6]
'''見出しと小見出し'''。見出しと小見出しの出現順序に関する制約はあるのでしょうか。

[9]
'''リストのレベル'''。
[CODE[-]] や [CODE[+]] なしで [CODE[--]] や [CODE[++]]
が現れたらどうするのでしょうか。 [CODE[-]] 系と [CODE[+]]
系の混在はどうなるのでしょうか。

[10] 
'''引用の出典の場所と個数'''。
[CODE[>:]] が一連の [CODE[>]] 行中に複数出現しても良いのでしょうか。
先頭以外でも良いのでしょうか。

[11]
'''引用の出典の書式'''。

> 形式は[URL]でなくてもかまいません。

他に何が認められるのでしょうか。段落内で認められるものがすべて認められるのでしょうか。

[12]
'''整形済みテキスト'''。
整形済みテキスト内でも外のように記法が認識されるのでしょうか。
されるとしたら、整形済みテキストを表す [CODE['''[''']]・・・[CODE[''']''']]
はどうなるのでしょうか。

[13]
'''表'''。
見出しを表す [CODE[*]] は表のどの[[こま]]でも使えるのでしょうか。
[CODE[|>]] や [CODE[|<]] が最初の行以外にあるとどうなるのでしょうか。
列のこま数がそろっていないとどうなるのでしょうか。
行末に [CODE[|]] がないとどうなるのでしょうか。

[14]
'''URL リンク'''。
リンクになる URL とはどんなものでしょうか。
[[RFC 3986]] の構文あたりでよいのでしょうか。
URL とは絶対 URI with optional 素片識別子と理解してよいのでしょうか。

[15]
'''[URL] リンク'''。
URL として [CODE['''[''']] や [CODE[''']''']]
を含めてもよいのでしょうか。

[16]
'''強調'''。単独の [CODE[!!!]] や [CODE[! !]] はどうなるのでしょうか。
[CODE[!!!]] の後に [CODE[!!!]] が来るよりも先に [CODE[! !]]
が来るとどうなるのでしょうか。同様に HTML タグを挟むとどうなるのでしょうか。

[17]
'''記法の停止'''。[[スペース]]は、データの一部なのでしょうか。

[18]
'''HTML タグ'''。まったくのチェックなしですべてなのでしょうか。
どの版の [[HTML]] に基づく構文でしょうか。
タグの不整合等はどうなるのでしょうか。
