*Chat で呼んでも返事してくれません。

まず、 Internet ってのは結構てきとーな機構なので、
あなたの発したメッセイジが即座に相手に伝わる保証なんて
どこにもありません。それはわかばにはどうしようもありません。

そういう状況がたまたま良かったとして、すぐに返答出来ないとしたら、
それはわかばが悪いのかもしれません。特に人が余りこない chat
では、画面に出したまま、他の仕事をしながら人を待つこともあります。
そういう時、あなたが呼びかけても、直ぐに気がつくとは限りません。
大らかな心をもって、待ってやって下さい。

わかばの使っている[RUBY[計算機] [コンピューター]]
が悪い事があります。若葉が悪名高い [[M$]] 社のソフトウェアに
悪戦苦闘しているため、返事したくても出来なかったり,
[[島流しの刑]](謎)になって復活するのに時間が掛かったりすることが
しばしばあります。そういう不良製品を使わなければ済む事ですが、
諸事情によりそう上手くはいきません。大らかな心で
許してやって下さい。

つまり正しいと思うものに従うってことです。

いまこうして使っているのは Windoze Me であります。そこらに転がってるのは Windoze 98
ですけど、あまり使いませんね。海^H廊下の向こうの部屋には
Windoze 2000 とか XP もあるので、たまに触らざるを得ません。
(触るのは大抵ネットワークがおかしいとか PowerPoint で
動画が再生できないとか困った時です(泣)。あたしゃ support
要員じゃないんじゃが。)

で、自分の機械では、 Windoze Me とはいえ、結構しばしば
DOS 窓開いて作業してます。一々 current directory を移るのが
面倒なので、 explorer でフォルダを右 click したら
そこを current にして DOS 窓が開くようにしています。 (Windoze 95
の治世でいうところの、 PowerToys (に入ってたっけ?) の

で作業することもしばしば。ちょっと前までは M$-DOS の
バッチファイルも使ってましたが、最近は Makefile にしてます。
shell script とか sed とか awk も使えるといいなあと思いますけど、
不勉強なので使えませんねえ。専ら Perl です。

それから、結構 suika.fam.cx に ssh で入って作業してます。
それの OS は GNU/Linux です。こういう分散計算環境では「使っている OS」
ってのはどっちになるのか決めかねますねえ。 (確かに、直接
触っている OS は Windoze ですけどね。)
