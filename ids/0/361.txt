* 記法

[5] ビット組合せの表現方法として [[ISO/IEC]]
の規格では [CODE[[VAR[x]]/[VAR[y]]]] という記法を採用しています。
[CODE[[VAR[y]]]] は下位4ビットの十進表現で、
[CODE[[VAR[x]]]] は残りのビットの十進表現です。

仕様書: 
-[[JIS X 0202]]:1998 ([[IDT]] [[ISO/IEC 2022]]:1994)
--[CSECTION[5. 記法, 符号表及び名前]]

[6] '''ビット組合せと符号表''':
[[JIS X 0202]]:1998 5.1 ではビット組合せの表現方法が規定されており、
これは[[符号表]]の位置の表現方法と同じだとされています。
同 5.2 では[[符号表]]の位置の表現方法が規定されており、
これはビット組合せの表現方法と同じだとされています。

[7] '''先導 [CODE[0]]''':
歴史的には色々ありますが、 [[JIS X 0202]]:1998
では次のようになっています。
- 5.1 のビット組合せの表現方法では、
[CODE[[VAR[x]]]] と [CODE[[VAR[y]]]]
は [Q[0〜15 の整数とする]]とされています。
- 5.2 の符号表の位置の表現方法では、
[Q[列番号及び行番号は、先頭の 0 を含めて表現する (例えば、 [SAMP[02/01]])]]
と述べられています。
- 5.1 と 5.2 で、2つの表現方法は同じものであるとされています (>>6)。
- 規格表全体を通じて、 [CODE[[VAR[x]]/[VAR[y]]]] 
表記はすべて先頭に 0 を含めて表現されています。
