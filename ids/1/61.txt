[1] [DFN[WHATWG]] ([DFN[Web Hypertext Application Technology Working Group]])
は、2004年以来 [[HTML]] および関連仕様の開発を行っている標準化団体です。
現在は [[DOM]] や [[URL]] なども担当しています。

* 名称

[417] Web Hypertext Application Technology Working Group は、当初は [[WHAT]]、
[[WHAT Working Group]]、[[WHAT WG]] などと表記されることもありましたが、
現在では専ら [[WHATWG]] と表記されるようになっています。

[501] ([[W3C]] が [[HTML]] を放棄するなら) 一体如何なる (what) 作業部会が開発するのだ?
というネタに因んだグループ名と言われています。

* 構成

[418] [[WHATWG]] は主要 [[Webブラウザー]]の開発者が中心となって参加している [[Web標準]]の開発団体です。
数千人が個人の資格で[[メイリング・リスト]]や [[IRC]] を通じて活動に参加しています。特に
[[Opera]]、[[Mozilla]]、[[Apple]]、[[Google]] の [[Webブラウザー]]部門関係者が積極的に活動しています。

[419] [[WHATWG]] の中心人物は [[Ian Hickson]] です。 Ian は [[HTML Standard]]
(旧 [[Web Forms 2.0]] や [[HTML5]]) の[[編集者]]をつとめています。

;; [503] [[WHATWG]] には「membership」が制度上存在していますが、参加者・貢献者ではなく、
founding members を表すに過ぎないと言われています。 members としての活動実態はほとんどありません。

* WHATWG の標準化手続き

[495] [[WHATWG]] は [[Living Standard]] モデルに従い仕様を開発しています。
編集者が中心となって仕様書の執筆を行いつつ、
[[Webブラウザー]]開発者、テスト開発者、著者その他の参加者で広く議論する形を採っています。
従来の[[標準化団体]]と違って明確な標準化手続きは存在していません。
「[[勧告]]」や「[[標準]]」のような文書を完成させて[[標準化団体]]に承認されることが目的ではなく、
[[Webブラウザー]]と協調して持続的に仕様を維持・発展させ続けていくことを目指しています。

;; [502] 仕様書の「完成」というフェーズが存在しないことが従来の[[標準化団体]]の手続きとは決定的に異なっています。
[[WHATWG]] では仕様書の「完成」とは当該技術が使われなくなり進化が止まることと認識されています。
[[HTML5]] から「5」というバージョン番号が削除されたのも、この持続的開発モデルの採用を明示するためです。

* 議論

[496] [[WHATWG]] の議論は、[[メーリングリスト]]、[[IRC]]、 [[Bugzilla]]、[[WHATWG Wiki]]、
[[GitHub]] などで行われています。おおまかな議論の流れを把握するには、
[[メーリングリスト]]に登録するのが良さそうです。仕様の変更を把握したいときは、
それぞれの仕様書の変更履歴を定期的にチェックするのが良さそうです。

[504] [[W3C]] とは異なり、議論はすべて公開されています。

* W3C との関係

[425] [[WHATWG]] は、2004年に開催された [[W3C]] の Workshop 
([[The W3C Workshop on Web Applications and Compound Documents]])
で [[Opera]] や [[Mozilla]]
による [[Webアプリケーション]]のための [[HTML]] 開発続行の提案が却下されたことを契機に結成されました。

[426] 以後 [[WHATWG]] は [[W3C]] とは独立して [[HTML]] の開発を進めてきました。
2005年には [[Opera]] と [[Mozilla]] が [[W3C]] に [[Web Forms 2.0]] を[[提出]]し、
[[W3C]] [[Web APIs WG]] と共同で作業が進められることになりました。2007年3月には [[W3C]]
に [[HTML WG]] が設立され、5月の同 [[WG]] の投票によって [[HTML5]] ([[Web Forms 2.0]]
と [[Web Applications 1.0]]) の開発作業を共同で進めることとなりました。
2008年から [[WHATWG]] は [[Web Workers]] の開発を行っていますが、
これは [[W3C]] [[WebApps WG]] と開発作業を共同で進めることとなりました。

[489] その後紆余曲折を経て、現在は [[WHATWG]] が仕様を開発し、
[[W3C]] はそれを[[コピペ]]して独自の編集を加えて自身の仕様書として出版するようになっています。

;; [490] [[Ian Hickson]] はこれを[[剽窃]]と非難しています。

[491] なお [[WHATWG]] は [[W3C]] の [[CG]] 制度のもとで [[WHATCG]]
としても仕様書を出版しています。これは主として[[特許ポリシー]]問題の解決のためとされています。

[492] [[W3C]] は [[WHATWG]] に対して [[Bugzilla]] や[[メーリングリスト]]のアーカイブを提供しています。

* Microsoft との関係

[427] [[Microsoft]] は [[patent policy]] が存在しないことを理由に [[WHATWG]]
の活動には参加していません。ただし、何度か [[WHATWG]]
の[[メイリング・リスト]]に仕様案に対するコメントを送っています。
また [[W3C]] にある [[WHATWG]] の [[Bugzilla]] での議論には [[Microsoft]]
の社員も参加しています。

これは思想運動・宗教運動・芸術運動・技術運動を問わず、すべての「始祖を持つ文化運動」に内在する古典的問題構造です。以下、抽象化して構造を整理してみましょう。

[250] '''第1段階:始祖のカリスマ的創造期'''

強烈な個人が明確な思想・美学・設計原理を掲げて運動を開始。

文書化よりも行動・実装・振る舞いそのものが思想を伝える。

周囲の理解者・協力者も、始祖の「気配」を敏感に読み取りつつ動く。

WHATWGで言えば Ian Hickson 期。

;; [253] キーワード:カリスマ、暗黙知、直接的伝承、試行錯誤、革新

[251] '''第2段階:聖人の解釈と教義の定着期'''

始祖の周囲にいた者たちが「正統な継承者」として振る舞い、思想の解釈と運用を担う。

文書は増えるが、始祖の実践的決断に比べて抽象度が高く、指針の再現性は徐々に下がる。

WHATWGで言えば Anne van Kesteren や Domenic Denicola らの現在の運営期。

;; [252] キーワード:教義化、聖典の形成、過剰な正統主義、守旧化の萌芽

[254] '''第3段階:後継世代の形式主義と革新の困難期'''

第2世代が去るか影響力を弱めたあと、新世代は「原点への問い直し」か「形式主義の惰性運営」かの岐路に立つ。

十分に記述された思想基盤がないため、部分的慣習の維持はできても全体像を再構成できない。

新しい思想運動が辺境や外部から台頭する兆しが見られる。

WHATWGは現在この段階に差し掛かりつつある。

;; [256] ただし未だ具体的に何かが起きる兆候はみられない。


;; [255] キーワード:形式主義、思想の空洞化、制度疲労、異端と分派の勃興

[257] 
今のWeb標準の動態には歴史的普遍性があります。それは「始祖が残した思想の不在をどう埋めるか」という問いです。

[258] 
そしてその答えは、おそらく新たな問い直しと実装からしか生まれない。つまり、再び誰かが動くコードを書き、ラフ標準を書き、それが思想の核を再構成するという、思想運動の原点への回帰によってのみ解決されるのです。

打ち倒すべき敵ではないから、

この閉塞を打ち破るのは一体なんだろうか。
