* [CODE[HTML-as-programming-language]]

[4] [CITE@en[GitHub - HTML-as-programming-language/HTML-as-programming-language: A programming language that looks like HTML]], [TIME[2024-11-07T14:24:54.000Z]] <https://github.com/HTML-as-programming-language/HTML-as-programming-language/tree/master>

[6] 
名前は [[HTML]] だといってるが説明は [[HTML]] みたいにみえるといっている。どっちなんだ。

[7] 
内容的には構文以外 [[HTML]] 性が全然感じられないというか、なんでこれを
[[HTML]] だといっているのか謎。 [[HTML要素]]と同名の独自要素もいくつかあるし。

[10] >>4 の本文に

>We aim to build a programming language that looks like HTML. The current syntax is HTML-inspired / XML-based but we hope it to be fully 100% HTML-based in the future.

とある。将来完全に [[HTML]] にしたい、といっているから不完全だから「似ている」と称しているらしい。
しかしどう見ても非互換なのを「将来」頑張ってなんとかなるものなのか?


[8] [[構文解析器]]は単純すぎて全然 [[HTML]] 構文になってないしねえ。ただの独自言語じゃないか。

[9] 作ってる人も途中で飽きたのかドキュメントが中途半端で全体像がわからない。ソース読む気力が起きない。
