[1] [CITE[ネットワーク層における情報検索機構の検討]]を読んだメモ。

- 背景
  - インターネット、特に WWW の普及
    - 情報量の急激な増加
    - 頻繁な情報の増減、移動
    -> 利用者の情報獲得は困難に
  - 情報の検索と転送の分離
    - 情報検索: アプリケーション層
    - 情報転送: ネットワーク層
- 問題
  - 情報検索結果の質が利用者の能力に大きく依存する
    - 得られる情報の質
    - ネットワーク的な近さ
      - c.f. キャッシュサービス
      - 検索サービス -> 集中的に管理するのは困難
  - 情報が頻繁に増減、移動している
    - 検索サービス -> 必ずしも最新の情報ではない
  <- 情報の検索と転送が独立している
- サービスモデル
  - ネットワーク層に「情報アドレス」を取り入れる
  - 動作イメージ
    - 利用者はホストアドレスの代わりに「Xが欲しい」を宛先とするパケットを送信する
    - ネットワークはこのパケットを情報Xを持つ計算機へ転送する
    - Xを持つ計算機が複数あれば、ネットワーク的に近い方へ転送する
    - ルータがXのキャッシュを持っていれば、パケットは転送せずにそれを返す
- 枠組み
  - 情報ネットワークアドレス
    - 情報の識別子
    - 情報と情報ネットワークアドレスの変換方法を決める必要がある
  - 情報ネットワークアドレスを用いたルーティング
    - 既存のルーティングと同様に実現できる
    - 情報ネットワークアドレス用の情報経路表を導入する
    - ネットワークアドレスのように集約できるとは限らない
    - 情報パケットの集約
      - 複数の計算機が同じ情報を返してきたら、どちらかを選ぶなどの処理が必要
  - 情報経路表の管理機構
- 実現方式
  - 情報
    - 任意長の文字列で表す
      - e.g. URL, Gコード, ISBN, JAN, バーコード
  - 情報ネットワークアドレス
    - IP アドレスを用いる
    - 変換
      - DNS のようなデータベースを用いる
      - ハッシュ関数等で算出する
        - ただし、逆変換できない
  - 情報ネットワークアドレスの共有
    - ルーティングプロトコルを用いる
    - 送信元アドレスを登録する
  - 情報ネットワークアドレス宛パケットの中継
    - 情報経路表に従い転送する
    - 経路表になければ
      - 情報ネットワークアドレスの性質により処理する
        - e.g. ブロードキャストアドレス -> すべてのリンク
      - 他のルータに問い合わせる
        - フラッディングのようなもの?
- 課題
  「このような提案は、現在の経路制御技術で緊急の課題となっている経路の爆発に対応し、
    ルーティング処理を単純化していくという動きに、ある意味では逆行するものであろう。
    しかしながら、情報ネットワークアドレスをネットワーク層で処理することでこれまでに
    ないネットワークサービスが実現できるならば、このような検討にも意義があると考える。」

* 

- 環境
  - IP
- 目的
  - 情報検索・転送
  - 高品質、高効率
- 名前
  - 「情報ネットワークアドレス」
    - 情報に IP アドレスを割り当てる
- 経路制御
  - IP の経路制御と同様
  - 別途、情報と情報ネットワークアドレスの対応付けが必要
- その他
  - パケット集約 (1つの問い合わせに対する複数の回答への対処)
